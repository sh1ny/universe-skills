## 4. Universe Validation Engine

- [x] 4.1 Add `validateUniverse(root)` exported function in `src/story.js` — resolves `universeRoot` via `resolveUniverseRoot(root)` (works from universe root or story root). If `universeRoot` is `null` and `root` is a story with a `universe` field in `story.md`, return immediately with a warning: `"Universe '<id>' not found — story works in standalone mode"` and `ok: true` (rule 4.6 — do NOT scan `null`). If `universeRoot` is `null` with no `universe` field, return `{ ok: true, errors: [], warnings: [] }` (no universe to validate). Otherwise, call `scanUniverse(universeRoot)` to get universe entities. If `root` contains a `story.md` (i.e. invoked from a story root), calls `scanProject(root)` to get story entities for cross-level checks (4.2, 4.4). If no `story.md` (invoked directly from universe root), cross-level checks are skipped — only universe-internal checks run (4.3, 4.5). Orchestrates five validation rules, returns `{ ok: boolean, errors: string[], warnings: string[] }`. Follow the pattern of `validateProject()` (line 263) and `validateLinks()` (line 330).
- [x] 4.2 Implement cross-level reference resolution (runs only when story context is available — invoked from a story root) — for each reference field in story entities that could point to universe-level entities (`character.relationships[].character`, `character.locations`, `location.notableCharacters`, `faction.members`, `faction.locations`, `artifact.owner`, `artifact.location`), build a transient combined map per entity type: `new Map([...storyIds, ...universeIds])`. Check each reference for existence in the combined map. Story-level takes precedence — if found at story level, universe level is not consulted. Existence-only check — NO backlink enforcement (unlike `validateLinks` which enforces bidirectional backlinks for same-level references like `character.locations` ↔ `location.notableCharacters`). Report error: `"Cross-level reference '<field>: <id>' does not resolve at story or universe level"`.
- [x] 4.3 Implement universe entity id validation — check all universe-level entity ids are kebab-case (reuse `isKebabId()`, line 1291) and unique within their entity type. Report errors: `"Duplicate entity id '<id>' in universe <type>"` and `"Universe entity id must be kebab-case"`.
- [x] 4.4 Implement no-shadowing validation (runs only when story context is available — invoked from a story root) — for each entity type present at both levels, check no id exists in both story-level `project.<type>` and universe-level `universeEntities.<type>`. Report error: `"Entity id '<id>' exists at both story and universe level — shadowing is not allowed"`.
- [x] 4.5 Implement `universe.md` frontmatter completeness check — read `universe.md` frontmatter and verify `name` and `schema-version` fields are present (reuse `requireScalar()` or `requireFields()`, line 2467). Report error: `"universe.md missing required frontmatter field: <field>"`.
- [x] 4.6 Implement universe resolution warning — if `story.md` has a `universe` field but `resolveUniverseRoot` returns `null`, emit warning: `"Universe '<id>' not found — story works in standalone mode"`. This is a warning, not an error — the story remains functional.

## 5. Universe Scan and Report

- [x] 5.1 Add `universeScan(root)` and `formatUniverseScan(result)` functions in `src/story.js` — `universeScan` resolves `universeRoot` via `resolveUniverseRoot(root)` (works whether `root` is a universe root or a story root), then calls `scanUniverse(universeRoot)` directly. Does NOT call `scanProject` — a universe root has `universe.md` but no `story.md`, so `scanProject` would fail. Returns compact list of `{ id, name, type, file }` per entity. `formatUniverseScan` formats as one-line-per-entity output (no body content). If `resolveUniverseRoot` returns `null`, report "No universe found". Follow the data-function + format-function pattern of `projectReport()`/`formatProjectReport()`.
- [x] 5.2 Add `universeReport(root)` and `formatUniverseReport(report)` functions in `src/story.js` — `universeReport` resolves `universeRoot` via `resolveUniverseRoot(root)`, calls `scanUniverse(universeRoot)` for entity counts, and calls `validateUniverse(root)` for validation issues. Produces `{ counts: { characters, locations, systems, factions, artifacts }, total, validation }`. `formatUniverseReport` outputs entity counts per type, total entity count, and any validation issues. Empty universe (freshly initialized, only `_index.md` files) reports zero for each type. If `resolveUniverseRoot` returns `null`, report "No universe found".
